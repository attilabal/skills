Business Subagent Orchestrator

A complexity gate for business work.

This skill helps an AI agent decide how much process a business task deserves: one pass, a reviewer pass, bounded subagent delegation, a small role team, or a larger workflow.

01

The mental model

The skill behaves like a pragmatic operations lead. It does not try to make every task look sophisticated. It chooses the minimum reliable operating model.

The question is not how many agents to run. The question is where fresh eyes, bounded workers, or a hard stop actually reduce risk.

Core principle
Gatekeeping

Mode first, output second

The agent chooses the work mode before producing the final material.

Proportionality

Escalate one level

When unsure, it becomes more careful without jumping straight to a full workflow.

Source discipline

Do not invent missing facts

Unsupported facts become questions, assumptions, risks, or constraints.

Output freedom

No fixed template library

The output shape follows the purpose, audience, and next action.

02

Runtime behavior

The skill makes a quick work-mode decision from available context. It asks for clarification only when the missing answer could materially change the mode, risk level, or final-use context.

1. Request

Complex business task

Documents, project notes, sales material, RFP input, account research, or executive work.

2. Triage

Measure the risk

Input volume, source quality, audience, cost of error, verifiability, and time pressure.

3. Mode

Visible choice

In the MVP, the agent states the selected work mode and one concise reason.

4. Work

Create or delegate

The lead agent may draft, review, delegate bounded work, or coordinate roles.

5. Gates

Final discipline

Source support, gaps, contradictions, overclaims, usability, and stop/go checks.

03

The five work modes

These modes form a ladder. The agent chooses the lightest safe mode, not the most impressive-looking one.

01

Single-pass

Fast internal draft, simple summary, low-stakes wording, or informal first opinion.

Signal: few inputs, low risk, speed matters.
02

Two-pass

Create first, then review for source support, tone, overclaims, gaps, and risk.

Signal: important, but not workflow-sized.
03

Cost-aware delegation

A high-reasoning lead agent delegates bounded, verifiable subtasks to cheaper workers.

Signal: processing volume without strategic judgment.
04

Small role team

Several perspectives matter: PMO, sales, legal, compliance, finance, communication, or QA.

Signal: perspective diversity matters more than volume.
05

Larger workflow

Many inputs, high stakes, multiple work packages, independent verification, and synthesis.

Signal: large document packs, RFPs, executive decisions.
04

Subagent delegation

A subagent is not a way to transfer responsibility. It is a way to process a bounded piece of work while the lead agent keeps ownership of synthesis and claims.

Lead agent

Strategy, boundaries, synthesis

  • chooses the work mode;
  • writes the worker mandate;
  • sets the verification level;
  • reconciles worker output with the full context;
  • owns or qualifies final claims.
SubtaskExactly what the worker should do.
Why delegableBounded, factual, checkable, low strategic judgment.
InputThe documents, notes, records, or sources provided.
Forbidden decisionsWhat the worker must not decide or imply.
Return formatStructured candidate input for the lead agent.
VerificationLevel 0-3 based on risk.
The lead agent should not choose a "smarter" or "dumber" agent. It should choose a work role that fits the risk, context, and verifiability of the subtask.
05

Worker verification ladder

Worker output is candidate input. Verification depth increases only when the subtask risk justifies it.

Level 0

Lead spot check

Low-risk extraction or classification that is factual and easy to check.

Level 1

One reviewer

Medium-risk worker output that affects final claims or external communication.

Level 2

Two verifiers

Higher-stakes output where error could create reputational, legal, financial, sales, or project risk.

Level 3

Fixer loop

Critical or disputed output. Fix issues found by verifiers, then re-verify.

After verification, the lead decides accept, revise, or reject. Checked output still does not become automatic truth.
06

Source gaps: stop or proceed?

The skill is strict about sources, but it does not require impossible completeness. The key question is whether the missing fact could materially change the safe use of the output.

stop

Ask for specific input

Use when the missing information could change the main conclusion and is realistically obtainable.

go with constraints

Proceed from what exists

Use when deadlines, sales pressure, or access limits require progress. State the main constraint.

source expansion

Search external sources

Use only when the missing information is realistically verifiable outside the private context.

07

Quality gates

These gates keep the system useful without forcing a fixed output template.

Source supportWhat is input-backed, externally verified, assumed, or inferred?
Source gapsCould a missing fact change the main answer?
ContradictionsDo sources conflict on dates, owners, scope, status, or terms?
OverclaimsDoes the wording sound stronger than the evidence?
Audience fitIs this internal, executive, client-facing, contract-related, or decision support?
UsabilityCan the user send, decide, brief, assign, or compare from this?
Worker relianceWas delegated output verified at the right depth?
ConstraintsAre limitations visible without overwhelming the output?
